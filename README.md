# Project scope
I know that is uncommon to find content creators who choose to have their content on personal blogs/websites. Usually, a go-to solution would be to use a CMS (Content Management System) like [WordPress](https://wordpress.com) or [Wix](https://www.wix.com).

However, building a web application with any of these technologies is restrictive due to their monolithic design. It’s also harder to manage and scale these kinds of applications, and also could get significantly slower.

I've come to have to chose between a dynamic site or a static one. As I said above, dynamic websites can be slower and harder to manage, to create a CSM involves a lot of effort, and time that I don't have.

So, I decided to use [hugo](https://gohugo.io) as a static site generator and [firebase](https://firebase.google.com) for database management.

# Programs and libraries hosted in this solution
## Hugo
To create a new site with hugo run the following command:
```
hugo new site site_name
```

## Node.js

Init nodejs
```
npm init -y
```

Install webpack & webpack-cli
```
npm i webpack webpack-cli -D
```

Insall firebase
```
npm install firebase
```
